# Теория по библиотеке Customtkinter 
--------------------------------------------------------------------------------
## Создание главного окна
Главное окно приложения можно создать при помощи класса **CTK()**. Также вы можете задавать для него некоторые свойства:

- размеры окна задаются в пикселях при помощи метода **geometry()**;
- заголовок окна можно изменять при помощи метода **title()**;
- цвет фона вы также можете изменить, используя параметр **fg_color** в методе **configure()**.

Пример:
```
root = ctk.CTk()
root.geometry('1200x700')
root.title('Название приложения')
root.configure(fg_color='lightgreen')
```

--------------------------------------------------------------------------------
## Создание простейших виджетов в главном окне
В главном окне можно создавать и размещать различные интерактивные элементы - тектовые поля, надписи, кнопки и т.д. Все эти элементы называются **виджетами**.

Виджеты в customtkinter создаются при помощи специальных классов, внутри которых указывается окно для размещения виджета в параметре **master**, а свойства для каждого виджета задаются в методе **configure()** с использованием некоторых параметров. Рассмотрим классы для создания простейших виджетов:

--------------------------------------------------------------------------------

1. Класс **CTkButton()** - используется для создания кнопки. Основные параметры:
- **text** - надпись на кнопке;
- **font** - шрифт;
- **fg_color** - цвет кнопки;
- **hover_color** - цвет кнопки при наведении курсора;
- **text_color** - цвет надписи на кнопке;
- **command** - функция, которая выполняется при нажатии на кнопку;
- **cursor** - форма курсора при наведении на кнопку (arrow - стрелка, cross - крестик, wait - знак ожидания, hand2 - классический);

и другие.

Пример создания кнопки:
```
btn = ctk.CTkButton(master=root)
btn.configure(text='Моя кнопка', command=check, font=my_font, fg_color='yellow', text_color='black', hover_color='orange', cursor='cross')
```

Шрифт можно создать при помощи класса **CTkFont()**. Дополннительные параметры - **weight** для жирного шрифта и **slant** для курсива:
```
my_font = font=ctk.CTkFont(family='Courier', size=20, weight='bold', slant='italic)
```

--------------------------------------------------------------------------------

2. Класс **CTkLabel()** - используется для создания надписей (тектовых меток) в окне. Основные параметры:
- **text** - текст надписи;
- **font** - шрифт;
- **text_color** - цвет шрифта;

и другие.

Пример создания надписи:
```
lbl = ctk.CTkLabel(master=root)
lbl.configure(text='Моя надпись', font=my_font, text_color='white')
```

--------------------------------------------------------------------------------

3. Класс **CTkEntry()** - используется для создания поля для ввода/вывода текста. Основные параметры:
- **font** - шрифт;
- **text_color** - цвет текста в поле для ввода;
- **justify** - расположение текста в поле (center - по центру, right - по правому краю, left - по левому краю);
- **state** - равен 'normal', если в поле можно вводить текст и изменять его, и 'disabled', если поле нельзя изменить;
- **placeholder_text** - подсказка, которая отображается внутри поля ввода серым цветом, пока поле пустое и не активное;

и другие.

Пример создания текстового поля:
```
entr = ctk.CTkEntry(master=root)
entr.configure(justify='center', state='normal', font=my_font, text_color='darkblue')
```

Некоторые методы текстового поля:

- метод **get()** - позволяет сохранить значение из поля в какую-нибудь переменную (пример: _s = entr.get()_ - для текста, _d = int(entr.get())_ - для целых чисел);
- метод **delete()** - позволяет удалить текст из тектового поля по заданным позициям (пример: _entr.delete(0, 'end')_ - очистка всего текста из поля полностью);
- метод **insert()** - вставляет новое значение в тектовое поле (пример: _entr.insert(0, 'новый текст')_).

## Размещение виджетов в главном окне
Элементы в главном окне размещаются при помощи метода **grid()**, в котором указывается номер строки и номер столбца - позиция ячейки, где виджет будет расположен. **Grid** разбивает окно на столбцы и строки, которые сужаются до минимума, когда они пусты, но адаптируются к размеру размещенных внутри них виджетов.

Сделаем так, чтобы наше окно состояло из 7 строк и 7 столбцов. Для этого каждой строке и каждому столбцу нужно задать "вес", равный единице:

```
r, c = 7, 7
for i in range(r):
    root.rowconfigure(i, weight=1)
for i in range(c):
    root.columnconfigure(i, weight=1)
```

Теперь мы можем располагать виджеты так, как нам удобно, при помощи метода **grid()**. Например, давайте создадим обычную кнопку и расположим её посередине окна, то есть в 3 строке и в 3 столбце:

```
btn = ctk.CTkButton(master=root)
btn.configure(text='Моя кнопка', command=check, font=my_font)
btn.grid(row=3, column=3, padx=10, pady=10, sticky='ew')
```

Параметр **row** отвечает за номер строки, **column** - за номер столбца. Параметры **padx** и **pady** задают отступы от элементов в соседних ячейках в пикселях. Параметр **sticky** растягивает виджеты по размеру ячейки('ns' - растяжение вверх и вниз, 'ew' - растяжене вправо и влево, 'nsew' - вовсе стороны).

## Выпадающие списки
Помимо кнопок, текстовых меток и текстовых полей, в главном окне можно создавать и другие более сложные интерактивные элементы.

Рассмотрим классы для создания выпадающих списков:

--------------------------------------------------------------------------------

1. Класс **CTkComboBox()** - используется для создания выпадающего списка. Вы можете выбирать элементы списка или вводить собственные значения в поле списка. Основные параметры:
- **values** - список элементов (строк), которые будут в выпадающем списке;
- **font** - шрифт;
- **fg_color** - цвет фона поля ввода;
- **dropdown_fg_color** - цвет фона выпадающего списка;
- **dropdown_font** - шрифт для элементов выпадающего списка;
- **button_color** - цвет кнопки-стрелки для открытия списка;
- **hover_color** - цвет кнопки-стрелки при наведении курсора;
- **text_color** - цвет надписи на выпадающем списке;
- **justify** - выравнивание (расположение) текста на выпадающем списке ('left', 'right', 'center');
- **state** - состояние списка (_normal_ - можно ввести своё значение или выбрать из списка; _readonly_ - нельзя вводить своё значение, но можно выбрать уже существующие; _disabled_ - списком невозможно воспользоваться);
- **command** - функция, которая выполняется при выборе нового элемента в списке (при создании функции учитывайте, что у неё есть аргумент, который нужно указать в скобках в её заголовке, - этот аргумент является выбранным элементом списка)

и другие.

**Важно!** Все эти параметры не являются обязательными, то есть их не всегда нужно использовать. Например, если в вашем приложении действие происходит при нажатии на кнопку, а не при выборе значения в списке, то параметр command указывать не нужно - он уже будет в свойствах кнопки. Или если вы хотите оставить для списка классическое оформление, то не нужно указывать все параметры для цветов: fg_color, button_color и т. д.

Пример создания выпадающего списка для простейшего калькулятора:
```
operations = ["Сложение", "Вычитание", "Умножение", "Деление"]
cmb = ctk.CTkComboBox(master=root)
cmb.configure(justify="center", values=operations, state="readonly", font=my_font, command=choice)
```

Некоторые методы выпадающего списка:
- метод **get()** - позволяет получить значение, выбранное в выпадающем списке, и либо сохранить его в какую-нибудь переменную, либо сравнить с каким-либо значением (пример: _s = cmb.get()_ - сохранение, _if cmb.get() == '...'_ - проверка на конкретное значение);
- метод **set()** - позволяет установить в списке значение по умолчанию, то есть это то значение, которое отобразится в списке при запуске программы (если не используете данный метод, при запуске в списке будет надпись CTkComboBox).

Пример использования set() для простейшего калькулятора (см. выше):
```
cmb.set("Сложение")
```

--------------------------------------------------------------------------------
2. Класс **CTkOptionMenu()** - также используется для создания выпадающего списка, но без возможности вводить в поле собственные значения. Вы можете только выбирать уже существующие элементы списка. Основные параметры:
- **values** - список элементов (строк), которые будут в выпадающем списке;
- **font** - шрифт;
- **fg_color** - цвет фона поля ввода;
- **dropdown_fg_color** - цвет фона выпадающего списка;
- **dropdown_font** - шрифт для элементов выпадающего списка;
- **button_color** - цвет кнопки-стрелки для открытия списка;
- **hover_color** - цвет кнопки-стрелки при наведении курсора;
- **text_color** - цвет надписи на выпадающем списке;
- **anchor** - выравнивание (расположение) текста на выпадающем списке (_'w'_ - по левому краю, _'e'_ - по правому краю, _'center'_ - по центру);
- **state** - состояние списка (_normal_ - можно ввести своё значение или выбрать из списка; _readonly_ - нельзя вводить своё значение, но можно выбрать уже существующие; _disabled_ - списком невозможно воспользоваться);
- **command** - функция, которая выполняется при выборе нового элемента в списке (при создании функции учитывайте, что у неё есть аргумент, который нужно указать в скобках в её заголовке, - этот аргумент является выбранным элементом списка)

и другие.

**Важно!** Все эти параметры не являются обязательными, то есть их не всегда нужно использовать. Например, если в вашем приложении действие происходит при нажатии на кнопку, а не при выборе значения в списке, то параметр command указывать не нужно - он уже будет в свойствах кнопки. Или если вы хотите оставить для списка классическое оформление, то не нужно указывать все параметры для цветов: fg_color, button_color и т. д.

Пример создания выпадающего списка для дней недели:
```
days = ["Понедельник", "Вторник", "Среда", "Четверг", "Пятница", "Суббота", "Воскресенье"]
week_menu = ctk.CTkOptionMenu(master=root)
week_menu.configure(values=days, font=my_font, command=choice, anchor='center', text_color='black', fg_color='mediumaquamarine', button_color='mediumseagreen')
```

Некоторые методы выпадающего списка:
- метод **get()** - позволяет получить значение, выбранное в выпадающем списке, и либо сохранить его в какую-нибудь переменную, либо сравнить с каким-либо значением (пример: _s = week_menu.get()_ - сохранение, _if week_menu.get() == '...'_ - проверка на конкретное значение);
- метод **set()** - позволяет установить в списке значение по умолчанию, то есть это то значение, которое отобразится в списке при запуске программы (если не используете данный метод, при запуске в списке будет надпись CTkOptionMenu).

Пример использования set() для списка с днями недели (см. выше):
```
week_menu.set("Дни недели")
```

## О параметре command
**command** — это параметр, который позволяет привязать какие-либо действия к виджету (например, к кнопке или к выпадающему списку). Эти действия прописываются в функции, которая автоматически вызывается, когда происходит определенное событие (для выпадающих списков — когда пользователь выбирает элемент, для кнопок - когда пользователь нажимает на кнопку).

Как это работает для разных виджетов:

1. **CTkButton** - функция срабатывает при нажатии на кнопку:
```
import customtkinter as ctk

def press():
    print("Кнопка была нажата!")  # Здесь можно выполнить любое другое действие: обновить другие виджеты, изменить цвет фона, изменить данные в текстовом поле и т.д.

root = ctk.CTk()

button = ctk.CTkButton(master=root)
button.configure(text="Нажми меня", command=press)  # функция press вызывается при нажатии
button.grid(row=0, column=0, padx=10, pady=10, sticky='ew')

root.mainloop()
```

2. **CTkOptionMenu** - функция сработает при выборе элемента из списка:
```
import customtkinter as ctk

def menu_func(choice):
    print(f"Пользователь выбрал: {choice}")  # Здесь можно выполнить любые другие действия: обновить другие виджеты, изменить цвет фона, изменить данные в текстовом поле и т.д.

root = ctk.CTk()

option_menu = ctk.CTkOptionMenu(master=root)
option_menu.configure(values=["Красный", "Зеленый", "Синий"], command=menu_func)  # функция menu_func получит выбранное в списке значение
option_menu.grid(row=0, column=0, padx=10, pady=10, sticky='ew')

root.mainloop()
```

3. **CTkComboBox** - функция также сработает при выборе элемента из списка, но не сработает, если пользователь введёт своё значение:
```
import customtkinter as ctk

def cmb_func(choice):
    print(f"Пользователь выбрал: {choice}")  # Здесь можно выполнить любые другие действия: обновить другие виджеты, изменить цвет фона, изменить данные в текстовом поле и т.д.

root = ctk.CTk()

cmb = ctk.CTkOptionMenu(master=root)
cmb.configure(values=["Красный", "Зеленый", "Синий"], command=cmb_func)  # функция cmb_func получит выбранное в списке значение
cmb.grid(row=0, column=0, padx=10, pady=10, sticky='ew')

root.mainloop()
```

## Удаление виджетов

**1. Метод destroy()** - полное удаление виджета. Этот метод удаляет без возможности восстановления, т. е. виджет больше нельзя будет отобразить - он исчезнет из памяти полнстью. Если захотите его вернуть, придётся создавать заново.

Пример:
```
...
lbl.destroy()  # удаляем надпись lbl из окна
...
lbl.grid(row=0, column=0)  # через некоторое время пытаемся вернуть
=> ошибка: bad window path name - такого виджета больше не существует!
```

**2. Метод grid_forget()** - не удаляет виджет полностью, а всего лишь скрывает его на время. После применения виджет не будет отображаться в окне, но всё ещё будет существовать в памяти => его можно будет отобразить снова.

Пример:
```
...
lbl.grid_forget()  # удаляем надпись lbl из окна
...
lbl.grid(row=0, column=0)  # через некоторое время пытаемся вернуть
=> надпись снова появляется на экране
```

**3. Удаление всех виджетов в окне** - полная очистка при помощи цикла for.

Иногда нужно очистить всё окно сразу. Для этого можно перебрать список всех виджетов главного окна и уничтожить их. Для получения списка всех виджетов вы можете воспользоваться методом **winfo_children().**

Пример:
```
...
# удаляем все виджеты из окна
for widget in winfo_children():
    widget.grid_forget()

lbl.grid(row=0, column=0)  # возвращаем один из виджетов
=> видим на экране один единственный виджет - надпись lbl
```

Обратите внимание: после destroy() виджеты перестают существовать, и если вы потом захотите создать новые, это будет обычное создание заново.

## Изменение виджетов в процессе работы приложения
В процессе взаимодействия пользователя с вашим приложением виджеты могут изменяться. Например, при нажатии на кнопку могут изменяться надписи (Label) в окне, или при выборе какого-либо элемента в одном списке могут изменяться элементы другого списка.

**Рассмотрим пример.** Требуется написать программу для ввода пользователем своего места проживания. 

Нам понадобится два выпадающих списка (ComboBox): для стран и для городов. А также тестовое поле (Entry) для ввода адреса проживания (улица, дом, квартира и т. д.). Причём список городов будет зависеть от страны, выбранной в списке стран. То есть нам нужно указать функцию в параметре command для списка стран и в этой функции проверять, какая страна была выбрана, а затем изменять элементы в списке городов. 

При нажатии на кнопку "Готово!" в окне нужно отобразить единственную надпись - готовый адрес (все остальные виджеты удаляются).

## Чекбоксы и радиокнопки

1. Класс **CTkCheckBox()** - класс для создания флажка или чекбокса.

- Позволяет пользователю выбрать или отменить выбор опции.
- Множественный выбор — можно отметить любое количество чекбоксов независимо друг от друга.

Основные параметры:

- **text** - надпись рядом с чекбоксом;
- **font** - шрифт надписи;
- **command** - функция, срабатывающая при выборе или отмене выбора чекбокса, т. е. при нажатии на него;
- **state** - состояние чекбокса (_normal_ - действующий; _readonly_ - только для чтения, нельзя сделать выбор; _disabled_ - заблокированный);
- **text_color** - цвет надписи рядом с чекбоксом;

и другие.

Методы чекбокса:

- **get()** - возвращает True (верно), если чекбокс отмечен, и False (неверно), если чекбокс не отмечен;
- **select()** - делает чекбокс отмеченным (устанавливает галочку);
- **deselect()** - делает чекбокс неотмеченным (снимает галочку);
- **toggle** - меняет состояние чекбоокса с отмеченного на неотмеченный или наоборот;

и другие.


Пример создания чебокса:
```
import customtkinter as ctk

def show_state():
    if cb.get():
        print("Флажок установлен")
    else:
        print("Флажок снят")

root = ctk.CTk()
cb = ctk.CTkCheckBox(master=root)
cb.configure(text="Нажми меня", command=show_state)
cb.grid()
root.mainloop()
```

2. Класс **CTkRadioButton()** - класс для создания радиокнопки или переключателя значений.

- Позволяет выбрать только один вариант из группы.
- Единственный выбор — при выборе нового варианта предыдущий автоматически снимается.

При работе с радиокнопками вы должны относить их к какой-либо группе, поскольку в одной программе могут быть радиокнопки с разным назначением: одни для смены оформления фона, другие для выбора возраста пользователя и т. д.
Все радиокнопки в одной группе должны использовать одну и ту же переменную (**variable**), но разные значения (**value**).

Значения радиокнопок могут быть строками, целыми числами или значениями True/False. В зависимости от выбранного вами типа значений создаётся переменная для данной группы кнопок, в которой задаётся начальное значение:

- для строк: **ctk.StringVar()**
- для целых чисел: **ctk.IntVar()**
- для True/False: **ctk.BooleanVar()**

Рассмотрим пример со значениями-числами:
```
import customtkinter as ctk

def show_choice():
    my_r = my_var.get()  # получаем значение выбранной радиокнопки
    print(f'Выбрана кнопка со значением {my_r}')

root = ctk.CTk()
my_var = ctk.IntVar(value=1)  # переменная для группы радиокнопок с целочисленными значениями (начальное значение = 1)

# создаём три радиокнопки со значениями 1, 2 и 3 (но надписи рядом с ними будут другие)
r1 = ctk.CTkRadioButton(master=root, value=1)
r1.configure(text="Номер 1", command=show_choice)
r1.grid()

r2 = ctk.CTkRadioButton(master=root, value=2)
r2.configure(text="Номер 2", command=show_choice)
r2.grid()

r3 = ctk.CTkRadioButton(master=root, value=3)
r3.configure(text="Номер 3", command=show_choice)
r3.grid()
root.mainloop()
```

Основные параметры:

- **text** - надпись рядом с радиокнопкой;
- **font** - шрифт надписи;
- **command** - функция, срабатывающая при выборе новой радиокнопки, т. е. при нажатии на неё;
- **state** - состояние радиокнопки (_normal_ - действующий; _readonly_ - только для чтения, нельзя сделать выбор; _disabled_ - заблокированный);
- **text_color** - цвет надписи рядом с радиокнопкой;
- **variable** - переменная, к группе которой принадлежит радиокнопка (в примере выше my_var);
- **value** - значение радиокнопки: строка или число (**value** указывается не в методе configure, а при создании кнопки в классе CTkRadioButton: смотри пример выше)

и другие.

Методы радиокнопки:

- **get()** - возвращает значение текущей выбранной радиокнопки;
- **select()** - делает радиокнопку выбранной;
- **deselect()** - делает радиокнопку невыбранной;

и другие.

## Фреймы

**Фрейм** — это своеобразный контейнер (рамка), который используется для группировки (объединения) других виджетов. Представьте себе прозрачную коробку: внутри неё можно хранить и размещать разные элементы (написи, кнопки, списки и т.д.).

**Если совсем просто:**
Представьте, что ваше окно — это комната, а фреймы — это полки, ящики и коробки в этой комнате.
Вместо того чтобы разбрасывать вещи (кнопки, надписи) по всей комнате, вы кладёте их в разные коробки. Так легче найти, легче переставить и легче наводить порядок.

**Зачем нужны фреймы?**

1. Чтобы группировать похожие элементы. Например, логично в приложении для списка дел размещать чекбоксы в отдельном фрейме, чтобы отделить их от других виджетов.
2. Чтобы легко перемещать группу элементов. Из примера выше, если все чекбоксы расположены внутри фрейма, то нам достаточно изменить положение фрейма, и чекбоксы сдвинутся вместе с ним, а если бы они были не внутри фрейма, то каждый пришлось бы двигать отдельно.
3. Чтобы не ломать голову с нумерацией. Номера строк и столбцов во фрейме считаются отдельно от главного окна. То есть, располагая элементы внутри фрейма, вы никак не повлияте на расположение элементов в самом окне. В нектором смысле, фрейм сам является окном внутри окна.
4. Чтобы красиво оформить приложение. Фрейм можно сделать цветным, с рамкой, со скруглёнными углами и толстыми границами. Так, в окне можно выделить цветом наиболее важную группу виджетов, чтобы глазу было легче ориентироваться.

Фрейм создаётся в программе при помощи класса **CTkFrame()**. При создании в скобках обязательно указывается окно, в котором фрейм будет расположен (**master**), а также ширина (**width**) и высота (**height**) фрейма в пикселях.

Основные параметры:

- **fg_color** - цвет фона фрейма;
- **border_width** - толщина границы (рамки) фрейма в пикселях;
- **border_color** - цвет границы фрейма;
- **corner_radius** - радиус скругления углов: от 0 до 180 (0 - квадратные углы, 10 - немного скругленные, 50 - сильно скглугленные, 180 - получите круглый фрейм);

и некоторые другие.

Методы фрейма:

- **grid_propagate(False)** - запрещает фрейму изменять размер (если не использовать, размеры фрейма будут подстраиваться под количество и размеры элементов внутри него);
- **winfo_children()** - возвращает список всех виджетов, созданных внутри фрейма;
- **grid_forget()** - скрывает фрейм и все виджеты внутри него ( в окне его не будет видно);
- **destroy()** - полностью удаляет фрейм и всё его содержимое из программы;
- **cget()** - позволяет получить значение параметра, указанного в скобках (например, сохраним текущий цвет фона фрейма frm в переменную color: _color = frm.cget("fg_color")_);

и другие.


Пример создания квадратного фрейма с размерами 300х300 пикселей:
```
import customtkinter as ctk

...

frm = ctk.CTkFrame(master=root, width=300, height=300)
frm.configure(fg_color='lightblue', border_color='darkblue', border_width=3, corner_radius=0)
frm.grid_propagate(False)
frm.grid(row=0, column=3, padx=10, pady=10)

...

root.mainloop()
```

## Дополнительные окна в приложении

**Главное окно** — это основа приложения. Оно создаётся один раз при помощи класса **CTk()** и существует всё время работы программы. Обычно в программе мы называем главное окно _root_.

**Дополнительные окна** — это окна, которые можно открывать поверх главного. Как правило, такие окна появляются при нажатии на кнопку. Они могут быть нужны, например, для:

- вынесения дополнительных функций в отдельное окно (настройки приложения, справка и т. д.);
- ввода данных (формы, анкеты, диалоги и т. д.);
- предупреждений и уведомлений;
- для выполнения нескольких действий в приложении одновременно (многозадачность).

Для создания дополнительных окон используется класс **CTkTopLevel()**.

Некоторые особенности и отличия дополнительных окон:

- Для появления дополнительного окна не нужно вызывать метод _mainloop()_. В отличие от главного окна в приложении, дополнительные окна отображаются автоматически.
- Дополнительных окон _CTkTopLevel()_ в программе можно создавать сколько угодно. Технических ограничений на количество окон нет (кроме памяти компьютера). При этом главное окно _CTk()_ должно быть единственным.
- Дополнительные окна не зависят от главного: их можно перемещать, сворачивать, закрывать отдельно от главного. Но если вы закроете главное окно приложения, все дополнительные закроются вместе с ним.

Основные параметры окон _CTkTopLevel()_:

- **fg_color** - цвет фона окна;
- **width** - ширина окна;
- **height** - высота окна.

**Примечание**. Обычно размеры окна задаются не при помощи параметров _width_ и _height_, а при помощи метода _geometry()_.

Основные методы окон _CTkTopLevel()_:

- **title()** - устанавливает заголовок окна (заголовок указывается как аргумент в скобках в кавычках);
- **geometry("WxH")** - задаёт ширину и высоту окна в пикселях (например, _window.geometry("1200x700")_);
- **resizable(w, h)** - разрешает или запрещает менять размеры окна по горизонтали (w) и по вертикали (h); если хотите запретить изменять размер окна, установите значение False (пример, _window.resizable(False, False)_);
- **minsize(W, H)** - устанавливает минимальные размеры окна в пикселях, до которых его можно уменьшить;
- **maxsize(W, H)** - устанавливает максимальные размеры окна в пикселях, до которых его можно увеличить;
- **iconify()** - сворачивает окно;
- **deiconify()** - разворачивает окно;
- **state()** - задаёт состояние окна (_normal_ - нормальное/_iconic_ - свёрнутое/_withdrawn_ - полностью скрытое (не отображается ни на экране, ни в панели задач)/_zoomed_ - развёрнуто на весь экран);
- **attributes('-topmost', True)** - отображает окно поверх других окон (если не используете, то главное окно будет расположено над дополнительным);
- **destroy()** - закрывает окно и удаляет его из программы;
- **winfo_exists()** - проверяет, существует ли окно (закрыто или не закрыто), возвращает True (существует) или False (закрыто);

и другие.

Пример создания дополнительного окна с фоном цвета "lightcyan" с размерами 700х500 (размер окна менять нельзя), расположенного поверх главного окна:
```
import customtkinter as ctk

root = CTk()

...

window = ctk.CTkToplevel(master=root)
window.title('Моё окно')
window.geometry('700x500')
window.resizable(False, False)
window.configure(fg_color='lightcyan')
window.attributes('-topmost', True) 

...

root.mainloop()
```

## Переключатель и ползунок

**1. CTkSwitch — переключатель (тумблер)**

**CTkSwitch** — это виджет, который напоминает физический переключатель (тумблер). Он может находиться в двух состояниях: включён (ON) или выключен (OFF).

Особенности:
- Выглядит как ползунок, который перемещается влево-вправо при нажатии на него
- Часто используется для включения/выключения чего-либо (например, звука)
- Визуально отличается от CTkCheckBox (чекбокс — квадратик с галочкой, переключатель — кружок на дорожке)

Основные параметры _CTkSwitch()_:

- **text** - текст рядом с переключателем;
- **command** - функция, которая будет выполняться при нажатии на переключатель (его включении/выключении);
- **fg_color** - цвет дорожки переключателя в выключенном состоянии;
- **progress_color** - цвет дорожки переключателя во включенном состоянии;
- **button_color** - цвет "кружка" переключателя;
- **button_hover_color** - цвет "кружка" переключателя при наведении курсора;
- **state** - состояние переключателя (_normal_ - нормальный, работает; _disabled_ - заблокирован, недействующий);
- **font** - шрифт текста рядом с переключателем

и другие.

Основные методы _CTkSwitch()_:

- **get()** - возвращает True (верно), если переключатель включен, и False (неверно), если выключен;
- **select()** - делает переключатель включенным;
- **deselect()** - делает переключатель выключенным;
- **toggle** - меняет состояние переключетеля с включенного на выключенный или наоборот

и другие.

Пример создания переключателя в зеленом стиле:
```
import customtkinter as ctk

def on_sound():
    value = sw.get()
    if value:
        print('Звук включен')
    else:
        print('Звук выключен')

root = CTk()

...

sw = ctk.CTkSwitch(master=root)
sw.configure(text='Включить звук', font=my_font, state='normal', fg_color='gray', button_color='green', progress_color='lightgreen', button_hover_color='darkgreen', command=on_sound)
sw.grid(row=0, column=0, padx=10, pady=10)

...

root.mainloop()
```

**2. CTkSlider — ползунок**

**CTkSlider** — это виджет, который позволяет пользователю выбирать числовое значение из диапазона, перемещая ползунок.

Особенности:
- Горизонтальная полоска с перемещаемым кружком
- Позволяет плавно менять значение
- Часто используется для регулировки громкости, яркости, размера и т.д.

Основные параметры _CTkSlider()_:

- **command** - функция, которая будет выполняться при перемещении ползунка (имеет начальное значение-параметр в скобках - значение, установленное на ползунке после перемещения);
- **from_** - первое значение ползунка (минимальное);
- **to** - последнее значение ползунка (максимальное);
- **number_of_steps** - количество возможных шагов ползунка;
- **fg_color** - цвет неактивной части ползунка;

и другие.

- **get()** - возвращает значение, установленное на ползунке;
- **set(value)** - устанавливает ползунок на значение value

Пример создания ползунка в красном стиле со значениями от 0 до 100 и с 10 возможными шагами:
```
import customtkinter as ctk

def my_func(value):
    print(f'Значение ползунка - {value}')

root = CTk()

...

sl = ctk.CTkSlider(master=root)
sl.configure(from_=0, to=100, number_of_steps=10, state='normal', fg_color='gray', progress_color='orange', button_color='red', button_hover_color='darkred', width=500, command=my_func)
sl.set(0)  # устанавливаем начальное значение 0
sl.grid(row=0, column=0, padx=10, pady=10)

...

root.mainloop()
```

## Метод after() - выполнение действий с задержкой

**after()** — это метод, который позволяет запланировать выполнение каких-либо действий (функции) через определённое количество миллисекунд. Он не блокирует программу, а работает в фоне, позволяя приложению не зависать, а работать в обычном режиме.

Синтаксис:
```
window.after(time, function)
```
Здесь _window_ - окно, в котором выполняются действия; _time_ - время, через которое выполнится функция (время задержки) в миллисекундах; _function_ - функция, которая должна выполниться после задержки.

**Простейший пример использования:**
```
import customtkinter as ctk

def say_hello():
    print("Привет через 2 секунды!")


root = CTk()

# Через 2000 миллисекунд (2 секунды) вызвать функцию say_hello
root.after(2000, say_hello)

root.mainloop()
```

Метод **after()** — это сердце анимаций, таймеров и периодических обновлений в приложениях с визуальным интерфейсом. Он позволяет:

- создавать таймеры обратного отсчёта;
- делать анимации и мигания;
- обновлять данные в реальном времени (часы, датчики);
- запускать задачи с задержкой

и многое другое.

**Один из самых частых примеров использования — это таймер, который обновляет надпись каждую секунду:**
```
import customtkinter as ctk

# функция для запуска таймера
def update_timer():
    global seconds
    # пока секунды не закончились, уменьшаем их количество на единицу
    if seconds > 0:
        seconds -= 1
        timer_label.configure(text=f"Осталось: {seconds} сек.")
        root.after(1000, update_timer)  # вызываем эту функцию снова через одну секунду
    # если секунды закончились, то выводим надпись "Время вышло!"
    else:
        timer_label.configure(text="Время вышло!")

root = ctk.CTk()
root.geometry("300x200")

seconds = 10
timer_label = ctk.CTkLabel(root)
timer_label.configure(text=f"Осталось: {seconds} сек.")
timer_label.grid()

# Запускаем таймер
update_timer()

root.mainloop()
```

Отменить повторяющиеся действия можно при помощи метода **after_cancel()**. Простейший пример:
```
import customtkinter as ctk

def say_hello():
    global after_id
    # выводим "Привет!" каждую секунду
    print("Привет!")
    after_id = root.after(1000, say_hello)

def stop():
    root.after_cancel(after_id)  # отменяем вывод
    print("Остановлено!")


root = ctk.CTk()

after_id = None  # переменная для повтора действий

btn_start = ctk.CTkButton(master=root)
btn_start.configure(text="Старт", command=say_hello)
btn_start.grid(row=0, column=0)

btn_stop = ctk.CTkButton(master=root)
btn_stop.configure(text="Стоп", command=stop)
btn_stop.grid(row=1, column=0)

root.mainloop()
```
Для большего понимания запустите программу на своём ноутбуке.

 ## Подключение звуковых эффектов с помощью библиотеки pygame

 **Pygame** — это библиотека для создания игр на Python, но мы используем только её **звуковую часть**, чтобы добавить в наши приложения звуковые эффекты.

**1. Установка и подключение библиотеки в программе**

Перед использованием функций и методов библиотеки её, как обычно, нужно установить через терминал (или через командную строку):
```
pip install pygame
```

Для работы со звуком нам понабится только один модуль из этой библиотеки - **mixer**. Поэтому в программе можно импортировать не всю библиотеку, а только один этот модуль:
```
from pygame import mixer
mixer.init() 
```

Как видите, помимо импорта модуля _mixer_, мы применили к нему метод **init()**. Этот метод _инициализирует_ звуковой модуль, то есть подготавливает компьютер (ноутбук) к работе со звуками. Строка _mixer.init()_, как и импорт библиотеки, является обязательной!

**2. Загрузка и воспроизведение звука**

В pygame можно использовать следующие форматы звуковых файлов:

- _.wav_ — самый надёжный, работает везде;
- _.ogg_ — тоже работает хорошо;
- _.mp3_ — может работать нестабильно на некоторых системах, но тоже подходит.

В pygame есть **два основных способа воспроизведения звука**, и их важно различать:

|Модуль|Для чего|Особенности|
